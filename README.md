# TP1_EDA
<p align="justify">
El programa permite a una compañía de turismo administrar reservas mediante una lista doblemente enlazada. Cada reserva guarda nombre del cliente, ID único y costo. Las operaciones disponibles son: insertar, buscar por ID, eliminar, actualizar y mostrar todas las reservas. Además, ofrece estadísticas como promedio, mínimo, máximo, mediana y moda de los costos, y permite ordenar la lista por nombre o por costo, en orden ascendente o descendente. Se eligió una lista doblemente enlazada porque facilita inserciones y eliminaciones sin depender de un tamaño fijo, evita desplazamientos como en arreglos y permite recorridos en ambos sentidos, lo que ayuda a calcular la mediana y reajustar punteros durante el ordenamiento. Aunque actualmente las inserciones se hacen al final, la estructura está preparada para futuras inserciones en cualquier posición. Los atributos de las clases tienen visibilidad de paquete para simplificar el acceso desde la lista, priorizando la claridad académica sin getters ni setters. Se incorporaron validaciones: si el costo es negativo, se corrige a cero y se notifica; antes de insertar, se verifica que el ID no esté duplicado. Para ordenar se usó merge sort (O(n log n)), estable y eficiente en listas enlazadas. La técnica de punteros lento y rápido encuentra la mitad sin conocer el tamaño. Al ordenar por costo, si hay empate se desempata por nombre, logrando un orden predecible. Tras cada ordenamiento, se actualiza la referencia a la cola (tail) porque la cabeza puede cambiar. Las estadísticas se obtienen recorriendo la lista una vez para suma, mínimo, máximo y conteo. La mediana asume que la lista está ordenada por costo (se advierte si no). La moda se calcula con doble bucle (O(n²)), suficiente para volúmenes moderados. El menú es un bucle con validación de entrada para evitar cierres abruptos. En cuanto a complejidad, la mayoría de las operaciones son O(n) por los recorridos; el ordenamiento es O(n log n) y la moda O(n²). Durante el desarrollo se realizaron mejoras manuales: validación de costo negativo también en actualizar, corrección de la mediana para listas pares, uso de compareToIgnoreCase en comparaciones de texto, creación del método actualizarTail() para mantener la coherencia tras ordenar, y controles para listas vacías y entradas inválidas, lo que hace el programa más robusto y usable.
El sistema incorpora una interfaz de línea de comandos (CLI) diseñada bajo un enfoque de robustez y experiencia de usuario, centralizada en un menú interactivo dinámico que gestiona el ciclo de vida de las reservas mediante la interacción directa con el operador. Un criterio de decisión fundamental fue permitir que el usuario ingrese manualmente todos los datos (nombres, identificaciones y costos), asegurando que el sistema procese información en tiempo real a través de capturas por teclado. La arquitectura del menú utiliza un ciclo de control do-while que garantiza la persistencia de la sesión, permitiendo realizar múltiples operaciones CRUD, análisis estadísticos y ordenamientos avanzados sin interrupciones. Para fortalecer este proceso, se implementaron validaciones preventivas mediante el método hasNextInt() y el manejo manual del búfer de Scanner, lo que asegura la resiliencia del programa ante entradas de datos inválidas y evita errores de sincronización al capturar cadenas de texto. Además, la lógica de interacción está desacoplada de la estructura de datos interna, enviando parámetros dinámicos hacia los métodos de ordenamiento Merge Sort según la dirección y el criterio elegidos por el usuario, lo que convierte a este menú en un controlador eficiente que garantiza la integridad de la Lista Doblemente Enlazada en todo momento.

# La Utilización De la IA:
<p align="justify">
Durante la construcción y el desarrollo del software se emplearon algunas de las herramientas de inteligencia artificial que existen como apoyo para generar la estructura base del código y refinar algunos algoritmos y conceptos relacionados con las listas doblemente ligadas para evidenciar una logica de programación adecuada. En el caso puntual de este ejercicio se realizaron varias mejoras de codigo para fortalecer la robustez y precision del programa, Primeramente se añadió la validación del costo en el metodo de actualización para evitar las incosistencias en los datos ingresados por el usuario, adicionalmente se revisaron las codificaciones relacionadas con los calculos estadisticos, Para las comparaciones de texto durante el ordenamiento, se unificó el criterio usando compareToIgnoreCase, eliminando problemas derivados de mayúsculas y minúsculas, tambien se creó el método actualizarTail() para recalcular la cola después de cada ordenamiento, ya que la cabeza cambia y la referencia a la cola quedaba desactualizada. Se agregaron comprobaciones de lista vacía en varios métodos para prevenir excepciones, y en el menú principal se incorporó validación de entrada con hasNextInt() para evitar cierres abruptos por caracteres no numéricos. No obstante también se hicieron ajustes en los punteros en los metodos de ordenación para garantizar que la doble dirección se mantenga correctamente Por último, se incluyeron mensajes informativos y advertencias para el usuario, como la notificación sobre la mediana cuando la lista no está ordenada, mejorando la transparencia y usabilidad del sistema.
